Getting Started

The following section will show you how to set up and use FoBo in your Lift project. The main documentation for general Bootstrap usage included in this demo is currently the Bootstrap-2.2.2 documentation.

Starter Project

Kick start! If you are about to start a new Lift project that includes FoBo with Bootstrap you can get a kick start by forking the Templating-With-Twitter-Bootstrap project, it uses the FoBo module pre-set-up for use with the latest release of Bootstrap. If you already have a Lift project that you want to extend with FoBo/Bootstrap functionality the next section is for you, so read on for setup information.

Module Setup

To integrate FoBo for usage in your existing Lift project this section will show you how to set it up. I am assuming you are using SBT but if you are using Maven or some other build system you will probably be able to extract the needed information, its pretty straight forward and easy.

If you are using Maven instead of SBT the FoBo artifacts is at the oss.sonatype.org repository.
The resolvers section:
For your convenience pre-compiled FoBo module artifacts currently resides at the same repository as Lift and other lift modules so there is no need for additional resolvers.
The library dependency section:
Declare the FoBo lift module line "net.liftmodules" %% "fobo" % (liftVersion+"-X.Y.Z-SNAPSHOT") withJavadoc() withSources() with the version string starting with the lift version number used for your project and appending the module version on the form "-X.Y.Z"

To initiate the FoBo module, chose the Toolkit option(s) you prefer and then call FoBo.init() from your lift projects Boot.

Using FoBo in your own project you should set up the init param for the toolkit module you need or if you are using the module defaults just call FoBo.init(). This demo uses the following unusually heavy module setup in lift Boot.

FoBo Bootstrap setup in Lift Boot 
   ⋮

/**
 * A class that's instantiated early and run.  It allows the application
 * to modify lift's environment
 **/
  class Boot extends Loggable {
    def boot {
           ⋮
      FoBo.InitParam.JQuery=FoBo.JQuery172  
      FoBo.InitParam.ToolKit=FoBo.Bootstrap220 
      FoBo.init()
           ⋮

Note the uniform setup without any version references in the html templates, making way for a smooth(er) toolkit upgrade. For more information see the FoBo modules Github.

Bootstrap Customization

For common use it is mostly sufficient to use the pre-compiled artifact but by using FoBo and the sub project FoBo-Less it is also possible to do heavy customization of bootstraps "Look&Feel" using less variables, however, this requires that you manage your own build of FoBo and FoBo-Less locally.

FoBo Snippets

The FoBo module comes with some convinience functions (snippets, optional to use) that will, among other things, help you, build menues, create popups and tooltips and more. For more information see the snippet package section in the FoBoAPI .

The FoBo.Navbar Snippet

The FoBo.Navbar snippet gives you a simple way to build your Bootstrap nav from Lift SiteMap item's, combine this with the use of the FoBo.TBLocInfo object and you can do pretty much anything you are used to only a lot easier and with a lot less code.

Navbar Usage 
					
 <span data-lift="FoBo.TBNavbar.builder?group=[LocGroup-Name]"></span>
 <span data-lift="FoBo.TBNavbar.builderPullRight?group=[LocGroup-Name]"></span>

The above shows you how to transform SiteMap items to nav items in a NavBar. You can specify one or more nav builder's, i.e groups of items, on a NavBar where each group is specified by it's Lift LocGroup belonging. Items in a builder group can be pulled to the right in the NavBar by using the pull right builder.

Navbar Example - The rendering
Navbar Example

This example represent a typical navbar that can be fixed to the top or the bottom of the page. It can also, like this one, be used with a reactive fluid grid container to make it scale down and be usefull on mobile devices and pads. In this example there are also menu dividers between the Home and Content items and in the dropdown menu between the bootstrap and FoBo documentation items, rendering of this decorators is acomplished by the use of the FoBo.TBLocInfo object in the SiteMap like FoBo.TBLocInfo.DividerVertical and FoBo.TBLocInfo.Divider respectively.

The FoBo.Navlist Snippet

The FoBo.Navlist snippet gives you a simple way to build a Bootstrap navlist from Lift SiteMap item's,

Navlist Usage 
					
 <span data-lift="FoBo.TBNavlist.builder?group=[LocGroup-Name]"></span>

The above shows you how to transform SiteMap items to list items in a Navlist.

Navlist Example - The rendering

This navlist example have some SiteMap menu link items and nav header's/menu lable's and a menu divider. The label's is set in Lift's SiteMap by "marking" the item as a FoBo.TBLocInfo.NavHeader item, The menu divider is "marked" by using FoBo.TBLocInfo.Divider in the SiteMap.

</html>

Set a popover and preventing default on-click action 
<!DOCTYPE html>
<html>
    ⋮
  <body">
           ⋮

      <script data-lift="FoBo.Bootstrap.popoverPreventDefault"></script>
           ⋮ 
  </body>        
</html>
