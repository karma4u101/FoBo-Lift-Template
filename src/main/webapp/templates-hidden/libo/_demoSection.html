The main documentation for general Bootstrap usage included in this demo is currently the Bootstrap-2.1.0 documentation.

Setup

To integrate FoBo for usage in your existing Lift project this section will show you how to set it up. I am assuming you are using SBT but if you are using Maven or some other build system you will probably be able to extract the needed information, its pretty straight forward and easy.

SBT setup for FoBo usage 
 :
 
resolvers ++= Seq(
   "Sonatype snapshots"             at "http://oss.sonatype.org/content/repositories/snapshots",
   "Sonatype releases"              at "http://oss.sonatype.org/content/repositories/releases",
   "Media4u101 Repository"          at "http://www.media4u101.se:8081/nexus/content/repositories/releases/",
   "Media4u101 Snapshot Repository" at "http://www.media4u101.se:8081/nexus/content/repositories/snapshots/"
    :
)
 
// Lift  
libraryDependencies ++= {
  val liftVersion = "2.5-M1" // Put the current/latest lift version here
  Seq(
    "net.liftweb"     %% "lift-webkit" % liftVersion % "compile->default" withSources(),
     :
    "net.liftmodules" %% "fobo"        % (liftVersion+"-X.Y.Z-SNAPSHOT") withJavadoc() withSources() 
    )
}                            

// Customize any further dependencies as desired
libraryDependencies ++= Seq(
  "org.eclipse.jetty"        % "jetty-webapp"   % "8.0.3.v20111011"  % "container",
   :
   )

In the resolvers section: -- For your convenience pre-compiled FoBo module artifacts currently resides at my Media4u101 repository so if you do not fork and compile the FoBo module(s) yourself you will need to add these lines and the module will be fetched for you from the repository (recommended).

In the library dependency section: -- Declare the FoBo lift module with the version string starting with the lift version number used for your project and appending the module version on the form "-X.Y.Z"

Example on github: -- Here is the complete build.sbt file used in Templating With Twitter Bootstrap

If you are using Maven instead of SBT the FoBo relevant resolvers is the Medai4u101 repository.

To initiate the FoBo module, chose the Toolkit option(s) you prefer and then call FoBo.init() from your lift projects Boot.

Using FoBo in your own project you should set up the init param for the toolkit module you need or if you are using the module defaults just call FoBo.init(). This demo uses the following unusually heavy module setup in lift Boot.

A bootstrap only setup could be done as a one-liner with net.liftmodules.FoBo.init()
FoBo Bootstrap setup in Lift Boot 
                            :
                           import net.liftmodules.FoBo

                           /**
                            * A class that's instantiated early and run.  It allows the application
                            * to modify lift's environment
                            **/
                            class Boot extends Loggable {
                              def boot {
                                  :
                                FoBo.InitParam.JQuery=FoBo.JQuery172  
                                FoBo.InitParam.ToolKit=FoBo.Bootstrap210 
                                FoBo.init()
                                  :

In Lift's html5 template:s add (depending on your setup) something like this:

Lift surround template 
                        <!DOCTYPE html>
                        <html>
                          <head>
                             :
                             <!--Main css for the Bootstrap module option-->
                             <link href="/classpath/fobo/bootstrap.css" type="text/css" rel="stylesheet">
                             <link href="/classpath/fobo/bootstrap-responsive.css" type="text/css" rel="stylesheet">
                             <!--app specific tweeks and overrides external to module goes here-->
                             <link href="[path to your projects app css]" type="text/css" rel="stylesheet">  
                             :                        
                          </head>
                          <!--Page setup with bootstrap scrollspy-->
                          <body data-spy="scroll" data-offset="40">
                             :
                             <!--In end of page (for fast pageload) add the scripts you need--> 
                             <script src="/classpath/fobo/bootstrap.js" type="text/javascript"></script>
                             :
                          </body>        
                        </html>

Note the uniform setup without any version references in the html templates, making way for a smooth(er) toolkit upgrade. For more information see the FoBo modules Github.

A minimal bootstrap only template setup would be to set the css path to "/classpath/fobo/bootstrap.css" and adding app. specific css and additional bootstrap scripts as needed.

FoBo Snippets

The FoBo module comes with some convinience functions (snippets, optional to use) that will generate some commonly used toolkit helper scripts for you. For instansce doing it the Lift - FoBo way you can simply do this using a FoBo:s snippet containing some bootstrap convinience functions:

The FoBo.Bootstrap Snippet
Activate dropdown 
                        <!DOCTYPE html>
                        <html>
                          <head>
                               :
                          <script data-lift="FoBo.ScriptHelper.registerLoadEventFactory"></script>         
                          <script data-lift="FoBo.Bootstrap.activateDropdown?on=.dropdown-toggle"></script>              
                          </head>
                           :       
                        </html>
Set a popover and preventing default on-click action 
                        <!DOCTYPE html>
                        <html>
                            :
                          <body">
                               :
	                      <a href="#" rel="popover" title="The Title" data-content="The content ..." id="theId">...</a>
                              <script data-lift="FoBo.Bootstrap.popover?id=#theId&options=placement:'left'"></script>
                              <script data-lift="FoBo.Bootstrap.popoverPreventDefault"></script>
                               : 
                          </body>        
                        </html>

.... instead of doing the following (a common bootstrap component init use case)

Transformed output 
                        <!DOCTYPE html>
                        <html>
                          <head>
                             :
		                  <script type="text/javascript">                                 		
			                function addLoadEvent(func) {
				              var oldonload = window.onload;
				              if (typeof window.onload != 'function') {
					            window.onload = func;
				              } else {
					            window.onload = function() {
						          if (oldonload) {
							        oldonload();
						          }
						          func();
					            }
				              }
			                }
			                addLoadEvent(function() {
				              $('.dropdown-toggle').dropdown()
			                });	   			               
		                  </script>                                 
                          </head>
                          <body">
                             :
                              <!-- Set a boostrap popover element -->   
	                      <a href="#" rel="popover" title="The Title" data-content="The content ..." id="theId">...</a>
                              <!-- Initiate the popover and place it on the left -->
	                      <script>
                                $(function () { 
                                  $("#theId").popover({placement:'left'});
                                }); 
                              </script> 
                              <!-- Prevent the default action for click on the popover -->
                              <script> 
                                $(function(){
                                  $("a[rel=popover]")
                                    .popover()
                                    .click(function(e) {
                                      e.preventDefault()
                                  });
                                });
                              </script> 
                               : 
                          </body>        
                        </html>
A Bootstrap tooltip 
                        <!DOCTYPE html>
                        <html>
                            :
                          <body">
                               :
                            This pages demonstrates a optional 
                            <a id="lift-tt" title="The most powerful, most secure web framework available today. It simply rocks!" href="http://liftweb.net/">Lift</a> 
                            module...   
                               :  
                            <script data-lift="FoBo.Bootstrap.tooltip?id=#lift-tt"></script>
                               : 
                          </body>        
                        </html>
I am using the html5 attribute "data-lift" for hocks to Lift but if you like you are wellcome to use the class attribute instead like class="lift:...."
The FoBo.BootstrapNav Snippet

The FoBo.BootstrapNav snippet will let you define lift menu parts that will be transformed into a Bootstrap Nav menu. This work is originally contributed to the FoBo module by community member Damian Helme, read his blog post Twitter Bootstrap Navbar Dropdowns and Lift's Menu.builder for more information.

Part of your Lift SiteMap with the LocGroup that will be transformed into Bootstrap Navbar 
                                  :
                                Menu("Test") / "test" >> LocGroup("test") >> PlaceHolder submenus (
                                  Menu("Test 2") / "test2",
                                  Menu("Test 3") / "test3"
                                ),
                                :
Your Lift Template 
                        <!DOCTYPE html>
                        <html>
                            :
                          <body">
                          
                           <div class="navbar">
                             <div class="navbar-inner">
                               <div class="container">                          
                                :  
                               <!--The navbar snippet invocation--> 
                               <span data-lift="FoBo.BootstrapNav.menuToTBNav?eager_eval=true"> 
                                 <span data-lift="Menu.builder?top:class=nav pull-right;li_item:class=active;linkToSelf=true;expandAll=true;group=test"></span>
                               </span>     
                                : 
                               </div>
                             </div>
                           </div>
                             :  
                          </body>        
                        </html>
In this demo you will find the rellevant code in the SiteMap section of Lift Boot and the snippet invoctation in the _embededTopbar-FoBo02.html template.
