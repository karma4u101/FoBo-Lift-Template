The FoBo Module and Bootstrap

The FoBo module gives your Lift project seamless front-end toolkit integration. This page focus on the FoBo/Lift Bootstrap v2.x/v3.x and includes general bootstrap usage and information on how to use some of FoBo's API functions.

FoBo v1.1 and above adds support for Bootstrap v3.x and there is now also a Lift v2.6 Template Applications - On Github that uses FoBo for it's Bootstrap v3.x and FoBo's BootstrapMegaMetaProtUser for ProtoUser views. See the slightly pimped live Demo and the FoBo's API for more information.

For general information about the FoBo module see the FoBo Demo Home.

Although the benefits of using the FoBo module is several without even using the FoBo Bootstrap API hooks (see demo ingress section), the sections below will take you through the FoBo module setup to enable Bootstrap in your Lift project and give you some clues on how you can make use of the API functions, to for instance, utilize Lifts SiteMap to populate bootstrap menus and more.

Although this page uses Bootstrap v2.x most of the information like setup, upgrade, customization applies to FoBo's Bootstrap v3.x support. FoBo's Bootstrap v3.x support extends the Snippet support with new features, see the API documentation and the Lift v2.6 Template Applications - Bootstrap v3.x Demo for more information.
